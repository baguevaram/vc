linkShaders

En la computación gráfica se usan los shaders como ya hemos visto en el curso para aplicar el sombreado a objetos en 3D, entendido como el debido nivel de luz, oscuridad y color, todo esto para mejorar la representación de un objeto 3D en un computador. La mayoría de estos corren en la GPU debido a la facilidad que tienen estas para procesar la geometría con mayor eficiencia.

linkLUMA

Se usó el método de Luma, con el objetivo de obtener una imagen convertida a su equivalente en escala de grises, esto se logró realizando el cálculo del promedio, pero lo realiza de forma ponderada para corregir la percepción con la que es percibida por los seres humanos.

prom=R∗0.2627+G∗0.678+B∗0.05933prom=\cfrac{R * 0.2627+G * 0.678+B * 0.0593}{3}prom=3R∗0.2627+G∗0.678+B∗0.0593​1
R=promR=promR=prom2
G=promG=promG=prom3
B=promB=promB=prom4

1linkprecision mediump float;

2linkuniform sampler2D texture;

3linkuniform vec3 weights;

4link// interpolated color (same name as in vertex shader)

5linkvarying vec4 vVertexColor;

6link// interpolated texcoord (same name as in vertex shader)

7linkvarying vec2 vTexCoord;

8linkvoid main() {

9link    vec4 pixelColor = texture2D(texture, vTexCoord);

10link    float mean = pixelColor.r * weights.x + pixelColor.g * weights.y + pixelColor.b*weights.z;

12link    pixelColor.y = mean;

13link    pixelColor.z = mean;

linkMascaras de convolución

En este ejercicio veremos la utilidad de aplicar convoluciones por medio de distintos kernels, estos se usan para alterar las imágenes y obtener alguna información extra o solo cambiar su apariencia. Para realizar estos cambios usaremos 3 kernels los cuales son matrices que multiplicarán los píxeles propios de la imagen. En hardware podemos aprovechar el poder de cálculo de las gpu para aplicar la convolución, para estos cálculos usamos los kernels como vectores y accedemos a los texeles propios de cada imagen con un fragment shader. En cada punto del slider podemos ver el resultado de aplicar 4 kernels, de izquierda a derecha son:

  • Identidad
  • Detección de bordes
  • Filtro Blur
  • Matriz de Sharpen

11link    // sharpen

12link    [0.0, -1.0, 0.0, -1.0, 5.0, -1.0, 0.0, -1.0, 0.0]

1linkprecision mediump float;

2linkuniform sampler2D texture;

3linkuniform float mask[9];

4linkuniform vec2 offset;

5link// interpolated color (same name as in vertex shader)

6linkvarying vec4 vVertexColor;

7link// interpolated texcoord (same name as in vertex shader)

8linkvarying vec2 vTexCoord;

9linkvoid main() {

10link    vec2 tex0 = vTexCoord.st + vec2(-offset.s, -offset.t);

11link    vec2 tex1 = vTexCoord.st + vec2(0.0, -offset.t);

12link    vec2 tex2 = vTexCoord.st + vec2(offset.s, -offset.t);

13link    vec2 tex3 = vTexCoord.st + vec2(-offset.s, 0.0);

14link    vec2 tex4 = vTexCoord.st + vec2(0.0, 0.0);

15link    vec2 tex5 = vTexCoord.st + vec2(offset.s, 0.0);

16link    vec2 tex6 = vTexCoord.st + vec2(-offset.s, offset.t);

17link    vec2 tex7 = vTexCoord.st + vec2(0.0, offset.t);

18link    vec2 tex8 = vTexCoord.st + vec2(offset.s, offset.t);

19link    vec4 rgba[9];

20link    rgba[0] = texture2D(texture, tex0);

21link    rgba[1] = texture2D(texture, tex1);

22link    rgba[2] = texture2D(texture, tex2);

23link    rgba[3] = texture2D(texture, tex3);

24link    rgba[4] = texture2D(texture, tex4);

25link    rgba[5] = texture2D(texture, tex5);

26link    rgba[6] = texture2D(texture, tex6);

27link    rgba[7] = texture2D(texture, tex7);

28link    rgba[8] = texture2D(texture, tex8);

29link    vec4 convolution;

30link    for (int i=0; i<9; i++){

31link        convolution+=rgba[i]*mask[i];

32link    }

33link    gl_FragColor = vec4(convolution.rgb, 1.0) * vVertexColor;

34link}
